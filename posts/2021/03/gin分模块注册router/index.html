gin分模块注册router

最基本的路由注册

下面是最基本的router注册方式,只适用于小型项目或学习demo

package main

import (
	"net/http"
	"github.com/gin-gonic/gin"
)

func helloHandler(c *gin.Context) {
	c.JSON(http.StatusOK, gin.JSON{
		"message": "Hello world!",
	})
}

func main() {
	r := gin.Default()
	r.GET("/hello", helloHandler)
	if err := r.Run(); err != nil {
		fmt.Println("launch service failed, err:%v\n", err)
	}
}

router注册拆出

当项目的规模逐渐变大时,在main,go中定义router非常不合适.此时就可以考虑将router部分代码单独写出。此时文件结构为

.
├── ...   --- 其他文件
├── main.go   --- 项目入口
└── router.go   --- router文件

其中router.go代码如下

package main

import (
	"net/http"
	"github.com/gin-gonic/gin"
)

func helloHandler(c *gin.Context) {
	c.JSON(http.StatusOK, gin.JSON{
		"message": "Hello world!",
	})
}

func InitRouter() *gin.Engine {
	r := gin.Default()
	r.GET("/hello", helloHandler)
	return r
}

此时main.go中调用上面定义好的InitRouter函数：

func main() {
	r := InitRouter()
	if err := r.Run(); err != nil {
		fmt.Println("launch service failed, err:%v\n", err)
	}
}

多个router文件

随着业务越来越多,我们需要把不同的业务代码分模块整理,此时就需要在app内定义不同模块的router，最后再统一注册. 我们可以定义一个app文件夹,专门做模块拆分.此时文件结构如下

.
├── ...   --- 其他文件
├── app
│   ├── blog
│   │   ├── handler.go   --- 逻辑处理
│   │   └── router.go   --- blog模块的router文件
│   └── user
│       ├── handler.go   --- 逻辑处理
│       └── router.go   --- user模块的router文件
├── main.go   --- 项目入口
└── routers
    └── routers.go   --- 项目的router文件

此时blog模块的router.go代码如下

func Router(e *gin.Engine) {
	// 为每个模块定义路由组
	blog := e.Group("/blog")
	{
		post.GET("/list", handler.GetPostList)
	}

}

此时user模块的router.go代码如下

func Router(e *gin.Engine) {
	user:=e.Group("/user")
	{
		user.GET("/list", handler.GetUserList)
	}

}

此时routers.go文件代码如下

// 部分已省略
type moduleRouter func(*gin.Engine)

var moduleRouters []moduleRouter

// 注册app的路由配置
func include(moduleRouter ...moduleRouter) {
	moduleRouters = append(moduleRouters, moduleRouter...)
}

func InitRouter() *gin.Engine {
	r := gin.New()
	// 加载多个模块的路由配置
	include(
		user.Router,
		blog.Router,
	)
	for _, moduleRouter := range moduleRouters {
		moduleRouter(r)
	}
	return r
}

main.go代码如下

func main() {
	// 初始化路由
	r := routers.InitRouter()
	if err := r.Run(); err != nil {
		fmt.Println("launch service failed, err:%v\n", err)
	}
}

如此一番，就完成了分模块router的定义与注册，这样做有以下两点优点

  1. 对项目来说,有利于项目横向发展
  2. 对开发人员来说,模块专人负责,代码改动影响范围小
